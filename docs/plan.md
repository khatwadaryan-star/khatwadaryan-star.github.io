# Project Plan
## Risk Analysis
### T - Transfer
- The GitHub website may crash, so I will instead transfer my work to GitHub Pages for reliable service.
### A - Avoid
- I may overcomplicate the website with advanced coding languages outside the scope, so I will avoid them and limit myself to the languages within the scope (Markdown, CSS, and HTML).
### M - Mitigate
- I may manage my time poorly, so I will mitigate this by following the WBS (Work Breakdown Structure), which breaks my work into small tasks to do at separate times.
### E - Eliminate
- I may have inconsistent design between my webpages, so I will eliminate this risk by using one shared external stylesheet (style.css).
## Task Schedule
### 1. Planning
- Define project scope (docs/scope.md)
- Identify risks and create task schedule (docs/plan.md)
### 2. Design
- Create homepage (index.html)
- Create about page (about.html)

- Apply consistent styling to both pages (style.css)
### 3. Aftermath

### 4. Submission
- Check all links to see if they work
